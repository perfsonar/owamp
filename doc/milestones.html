OWAMP Version History

http://e2epi.internet2.edu/owamp/
$Date$

This document is an attempt to outline the major changes associated with each new release. This is not a comprehensive list. For complete details please see the cvs log messages as archived in the Changes file in the distribution.

Version 3.1a - 15-Jan-2008

  • Minor protocol compliance fixes
  • RPM support
  • Documentation updates
  • Add packet reordering to -M output

Version 3.0c - 24-Feb-2007

  • Many documentation fixes
  • Added option to sub-set test results. See the new -N option of owping.
  • Fixed bugs in powstream relating to summary data not always being properly flushed before powstream printed the filename.

Version 3.0b - 02-Jan-2007

  • Fixes a bug in the end-of-test waiting code for the receiver

Version 3.0a - 05-Dec-2006

  • Now compliant with RFC4656
    • Standard port number for owamp-control: TCP/861
    • Minor protocol changes - encryption bits changed
  • Includes man-page for powstream

Version 2.0c - 25-May-2006

  • Fixed a bug in the statistics code that incorrectly magnified loss. (A single lost packet would cause a pattern of lost packets: one every 'timeout' later.)

Version 2.0b - 11-May-2006

  • Major enhancements to signal handling aspects of all applications in the distribution
  • Graceful termination of sessions added using signal handling enhancements
  • Man-page for owup(1) added

Version 2.0a - 23-Mar-2006

  • Added owup command-line utility
  • Fixed bug in statistics output (maxerr was reporting maxdelay)
  • Fixed bug in owampd server greeting message (incorrect start time was being reported)

Version 2.0 - 15-Mar-2006

  • Major changes to protocol - mostly in support of early termination of sessions
  • Additional statistics now reported: TTL, Delay Variation (Jitter)
  • NTP requirements relaxed
  • Additional system ports:
    • FreeBSD 4.7, 5.4, 6.0 (64-bit)
    • Linux 2.4.21 (32-bit), 2.6.11 (64-bit)
    • MacOS X 10.4.5
    • Solaris 5.8
  • Man-pages for all supported applications
  • Firewall options: Can specify port-ranges for tests
  • Many, many bug fixes
